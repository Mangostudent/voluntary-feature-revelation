CODE DIRECTORY

Place your machine learning scripts, data processing notebooks, and evaluation files here.

Suggested Structure:
- notebooks/: Interactive Jupyter Notebooks (.ipynb) for initial experiments, exploratory data analysis (EDA), and plotting results.
- src/: Clean, reusable Python scripts (.py) containing:
  - data.py: Preprocessing, loading, and batching code.
  - model.py: Model architecture definition.
  - train.py: Model training loop.
  - evaluate.py: Evaluation metrics, testing, and error analysis.
- requirements.txt: List of Python library dependencies (e.g., torch, scikit-learn, numpy, pandas) for easy replication.

Code Policy:
- Academic papers typically declare the availability of software code in the declarations or acknowledgments section.
- Clean code with documentation helps reviewers and improves citation count.
